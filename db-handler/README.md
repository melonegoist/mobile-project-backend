# Db-handler

Серверная часть торгового приложения для управления акциями, портфелем и пользователями. Проект предоставляет REST API для взаимодействия с биржевыми данными, выполнения сделок и управления счетом.

## Архитектура проекта

Проект построен с использованием многослойной архитектуры на базе Spring Boot:

- **Контроллеры (`controller`)**: Обрабатывают входящие HTTP-запросы, направляют их в сервисы и формируют ответы.
- **Сервисы (`service`)**: Содержат бизнес-логику приложения (обработка сделок, расчет портфеля, аутентификация).
- **Репозитории (`repository`)**: Отвечают за взаимодействие с базой данных через Spring Data JPA.
- **Сущности (`entity`)**: Объектно-реляционное отображение (ORM) таблиц базы данных.
- **Модели (`model`)**: Data Transfer Objects (DTO), сгенерированные из OpenAPI-спецификации, для обмена данными с клиентом.
- **Конфигурация (`config`)**: Классы конфигурации Spring (безопасность, бины).
- **Безопасность (`security`)**: Компоненты для JWT-аутентификации и авторизации.
- **Утилиты (`util`)**: Вспомогательные классы (например, `SecurityUtils` для получения текущего пользователя).

## API Документация

API соответствует спецификации OpenAPI 3.0. Полная документация доступна в файле `resources/openapi.yaml` или после запуска приложения через Swagger UI (если настроен).

Базовый URL: `http://localhost:8080`

### Аутентификация

Большинство эндпоинтов требуют JWT-токен, который необходимо передавать в заголовке `Authorization: Bearer <token>`.

| Метод | Эндпоинт            | Описание                            | Тело запроса               | Ответ              |
| :---- | :------------------ | :---------------------------------- | :------------------------- | :----------------- |
| POST  | `/auth/register`    | Регистрация нового пользователя     | `RegisterRequest`          | `LoginResponse`     |
| POST  | `/auth/login`       | Вход в систему                      | `LoginRequest`             | `LoginResponse`     |
| POST  | `/auth/refresh`     | Обновление пары токенов              | `RefreshTokenRequest`      | `LoginResponse`     |
| POST  | `/auth/logout`      | Выход из системы (инвалидация токена)| -                          | `{ message: ... }`  |

## База данных

В проекте используется PostgreSQL. Управление схемой данных осуществляется с помощью **Flyway**.

### Схема данных

Основные таблицы:
- `users` — информация о пользователях.
- `accounts` — счета пользователей (баланс, валюта).
- `stocks` — справочник доступных акций.
- `portfolio_items` — текущие позиции пользователей в портфеле.
- `trades` — история совершенных сделок.
- `price_history` — исторические данные о ценах (свечи).
- `account_transactions` — история пополнений и выводов средств.
- `watchlist` — списки наблюдения пользователей.
- `user_settings` — настройки пользователей.
- `refresh_tokens` — refresh-токены для обновления JWT.
- `user_stats` — агрегированная статистика по торговле.

### Миграции

Скрипты миграции находятся в `resources/db/migration/`:
- `V1__init.sql` — создание всех таблиц, индексов и триггеров.
- `V2__seed_stocks.sql` — наполнение таблицы `stocks` начальными тестовыми данными.

## Безопасность

- **Аутентификация**: Реализована на основе JWT (JSON Web Tokens).
- **Хеширование паролей**: Используется `BCryptPasswordEncoder`.
- **Доступ к эндпоинтам**:
  - Публичные: `/auth/**`, `/health`.
  - Все остальные эндпоинты требуют валидный JWT-токен.
- **Токены**:
  - **Access Token**: Короткоживущий (по умолчанию 1 час) для доступа к API.
  - **Refresh Token**: Долгоживущий (по умолчанию 24 часа), хранится в БД и используется для получения новой пары токенов.
- **Контекст безопасности**: ID текущего пользователя извлекается из токена и доступен через `SecurityUtils.getCurrentUserId()`.
